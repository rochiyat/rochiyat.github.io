üìå Jadi gimana sih cara nulis dokumentasi yang enak dibaca dan gak bikin ngantuk? Ini mungkin bisa jadi cara sederhana dan realistis ala programmer:

1. README Adalah Gerbang Utama

Isi minimal:

  • Apa project ini?

  • Gimana cara install dan run?

  • Teknologi yang dipakai?

  • Struktur folder dan file penting.

Anggap aja kamu lagi ngenalin rumah ke tamu baru. Jangan biarin mereka nyasar ke gudang duluan.

2. Comment Seperlunya

Bukan semua baris dikasih comment. Tapi bagian:

  • Logic kompleks

  • Shortcut atau asumsi khusus

  • Hal yang bisa bikin developer lain (atau kamu sendiri) bingung nanti

3. Catat Hal-hal di Luar Kebiasaan

  • Harus pakai env tertentu

  • Port khusus

  • Library versi non-standar

  • Cara setup yang beda dari biasanya

Anggap aja kamu lagi nitip rumah ke teman, dan kamu kasih tahu: ‚ÄúOh iya, pintunya agak macet ya, mesti didorong pakai bahu.‚Äù

4. Gunakan Tools Kalau Bisa

Mungkin bisa coba pake:

  • Swagger buat dokumentasi API

  • JSDoc / TSDoc buat penjelasan fungsi di code

  • Markdown buat catatan ringan

  • Notion atau Confluence untuk wiki project internal

Nggak perlu pakai kalimat berbunga. Cukup jelas, ringkas, dan fokus bantu orang ngerti why dan how. Dokumentasi itu kayak peta harta karun. Bukan yang bikin kita tersesat, tapi yang bantu kita sampai ke tujuan.
